# 证据流 Agent PRD

## 1. 产品概述

- 产品名称：证据流 Agent（EvidenceFlow Agent）
- 赛道：赛道五，办公助手 Agent
- 产品定位：面向科研组会场景的行动转化 Agent，将“学生本周汇报 + 导师点评 + 新 idea”转化为“下周可执行研究计划、推荐阅读与交付物”。
- 核心价值：帮助学生把导师的话转成下周能做的事，帮助导师把组会真正推进成研究执行。

## 2. 背景与问题

目标用户在科研组会和课程研究型项目推进中，普遍存在以下问题：

- 学生每周汇报很多，但“做了什么、做到哪一步、卡在哪里”缺少统一结构。
- 导师现场给出的新 idea 往往停留在口头建议，几天后就模糊或走样。
- 下周要做什么常常不够具体，没有实验动作、阅读清单、验证指标。
- 研究历史沉淀差，导师和学生都很难持续追踪某个方向试过什么、为什么换方向。
- 周报、组会 briefing、汇报大纲重复整理，机械劳动重。
- 学生回到工位后往往仍然不清楚“先做哪个实验、先读哪篇论文、下次组会要回答什么问题”。

现有工具多停留在“纪要生成”或“文档问答”，缺少从“周进展汇报”到“下周研究执行”的闭环能力。

## 3. 产品目标

### 3.1 MVP 目标

- 支持导入会议录音或 transcript。
- 自动提取学生本周进展、结果、阻塞点、风险和未决问题。
- 自动捕获导师新 idea，并转成可执行研究计划。
- 自动生成下周行动清单、推荐阅读和验证指标。
- 自动生成周报、下次组会 briefing、下周研究计划和汇报大纲。
- 在必要时补充相关论文和参考依据。
- 沉淀按学生和项目维度的长期研究记忆，供后续组会复用。

### 3.2 非目标

- 不做企业级多租户协作平台。
- 不做实时会议 bot 接入。
- 不做完整 PPT 可视化编辑器。
- 不做复杂 IM、邮箱、日历集成的第一版。
- 不做自动实验执行系统或自动跑模型平台。

## 4. 用户角色

- 学生：需要把导师口头建议转成可执行的实验计划和阅读任务。
- 导师 / PI：需要快速理解学生本周进展、问题与下一步方向。
- 课题组负责人 / 组会组织者：需要沉淀项目历史、减少重复汇报成本。

## 5. 关键场景

### 场景 A：学生周进展结构化

用户上传录音或 transcript，Agent 输出：

- 本周完成项
- 当前结果
- 阻塞点
- 风险点
- 未解决问题

### 场景 B：导师新 idea 捕获与落地

针对导师在组会中的建议，Agent 自动输出：

- 新研究假设
- 建议实验动作
- 推荐阅读
- 预期验证指标
- 下周执行清单

### 场景 C：下周行动生成

针对本次组会内容，Agent 自动输出：

- 下周优先任务
- 建议先做的实验
- 建议先读的论文
- 下次汇报要回答的问题

### 场景 D：结论与 idea 参考依据补全

针对会议中的关键判断句，Agent 自动检索论文或项目文档，输出：

- 支持证据
- 反驳证据
- 待验证结论

### 场景 E：下次组会准备

在下次组会前，Agent 自动生成 briefing：

- 上次组会承诺回顾
- 未完成研究动作
- 即将到期任务
- 当前最大风险
- 导师最该关注的问题
- 建议议程

## 6. 产品能力分层

### P0：必须实现

- 会议内容导入与本地转写
- transcript 结构化抽取
- 学生周进展提取
- 导师 idea 捕获
- 下周研究计划生成
- 推荐阅读生成
- 学生 / 项目长期记忆
- 周报 / briefing / 汇报大纲生成

### P1：高价值增强

- 实验优先级建议
- 下次汇报问题清单
- transcript 可追溯回跳
- 学生研究轨迹摘要
- 缺证据结论 / idea 哨兵

### P2：比赛加分项

- 证据补全与引用卡片
- 证据冲突提示
- 导师质询 / 答辩陪练模式
- 知识地图 / 证据图谱
- 任务依赖可视化
- 主动提醒与阻塞预测
- 学生维度的研究方向演化图

## 7. Agent 设计

- 主控 Agent：理解目标，调度其他能力。
- 记忆管家 Agent：维护学生、项目、论文和历史决策记忆。
- 推进 Agent：将导师 idea 和会议结论转成下周研究计划及交付物。
- 推荐阅读 Agent：根据新 idea 和当前阻塞点推荐相关工作与阅读入口。
- 证据猎手 Agent：在需要时执行文献检索、引文追踪和证据聚合。

## 8. 成功指标

- 周进展抽取准确率达到可演示水平，人工修正率低于 30%。
- 至少 80% 的导师 idea 能生成结构化下周行动建议。
- 至少 80% 的下周计划附带推荐阅读或验证指标。
- 一次完整 demo 在 5 分钟内跑通。
- 用户可在一个页面完成“导入 -> 核验 -> 交付”主流程。

## 9. MVP 验收标准

- 能导入一段组会内容并生成结构化结果。
- 能为至少 1 名学生生成下周研究计划。
- 能为至少 1 名学生生成推荐阅读与验证指标。
- 能保存学生 / 项目上下文并用于下一次 briefing。
- 能导出 Markdown 格式的周报、下周计划和汇报大纲。
